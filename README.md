# 統合業務ツール（Chrome拡張機能）
## 解決した課題
| Before | After |
|--------|-------|
| 部署内の異なるチーム（プロダクト）ごとにマニュアルやツールが乱立 | 1つの拡張に統合 |
| 拡張機能を活用したいが数が増えると切り替えにも手間がかかってしまう | メニューからスムーズに選択できる |

## 機能について
- まったく異なる機能を一つの拡張機能内で利用可能
- トップ画面からチーム（プロダクト）を切り替えて、それぞれ異なるメニューの利用が可能
- 今後ひな型に倣って機能を追加可能

## 実装したツールの例
### ナレッジベース
- カテゴリ階層、検索、HTML形式対応
- スプレッドシートをデータベースとするため誰でも追加・編集が可能
- 属人化を防止
- メール対応時間1件あたり平均2分ほど削減。年 **200時間/人** ほどの時間短縮

- 審査フローや結果を入力するためのテキスト自動生成機能
- ミスの防止および年 **20時間/人** ほどの時間短縮

# Zendeskデータエクスポーター（Python / GUI）
## 解決した課題
| Before | After |
|--------|-------|
| 毎週手動でZendesk管理画面からデータをエクスポート | ワンクリックまたは自動で出力 |
| 複数ブランドのデータを個別に取得して手作業で統合 | ブランド選択して一括エクスポート |
| 日別・ブランド別の統計を手計算 | 問い合わせ件数・UUを自動集計、KPIへの転記 |
| 別プロダクト用のZendeskでは使用できない | UI上で認証してどのZendeskでも汎用的かつ直感的に使用可能 |

## 機能について
- Zendesk APIからチケットデータを取得し、Excel形式でエクスポート
- 認証情報の暗号化保存
- ブランド / カラム / 除外タグを選択可能
- AIエージェント対応チケットを別シートに分離するオプション
- 日別統計シート（UU数・回答数）の自動生成
- タスクスケジューラによる週次自動実行対応
- 古いエクスポートファイルの自動アーカイブ（14日経過でoldフォルダへ）
- レポート作成の完全自動化により **30分/週** の作業を削減

# シフト表Webアプリ（Google Apps Script）
## 解決した課題
| Before | After |
|--------|-------|
| 各メンバーのGoogleカレンダーを個別に開いて予定を確認 | 1画面でチーム全員の予定を一覧表示 |
| シフト表とGoogleカレンダーの二重管理 | 1画面で確認可能 |
| チーム内の連絡事項が分散 | 掲示板機能で当日の連絡を一元化 |

## 機能について
- Googleカレンダーから5営業日分の予定を自動取得・表示
- メンバーリストはスプレッドシートで管理（追加・削除が容易）
- チームカレンダーで各メンバーへの予定を入力
- 掲示板機能（投稿・編集・削除・ピン留め、日付変更でリセット）
- Slack通知機能（更新をワンクリックでチームに通知）
- キャッシュ機能で高速表示、バックグラウンドで最新データをチェック
- 予定指示・確認の時間を1日あたり **5分/人** ほど短縮
