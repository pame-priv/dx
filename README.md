# 統合業務ツール（Chrome拡張機能）

## 解決した課題

| Before | After |
|--------|-------|
| 部署内の異なるチーム（プロダクト）ごとにマニュアルやツールが乱立 | 1つの拡張に統合 |
| 拡張機能を活用したいが数が増えると切り替えにも手間がかかってしまう | メニューからスムーズに選択できる |

## 機能について
 - まったく異なる機能を一つの拡張機能内で利用可能
 - トップ画面からチーム（プロダクト）を切り替えて、それぞれ異なるメニューの利用が可能

  
### 設定駆動
```javascript
// これだけで業務を追加できる
{ id: "business-b", name: "業務B", configPath: "業務B/menu_config.js" }
```
コードを書かずに業務・メニューを追加できる設計。

### 遅延読み込み
選択したメニューのみ読み込むため、メニュー数が増えてもパフォーマンスに影響しない。

### パス自動解決
```javascript
const basePath = location.pathname.substring(0, location.pathname.lastIndexOf('/'));
```
コピー＆ペーストだけで新しい業務を追加可能に。

## 主な機能

- **マルチ業務対応** - タブで業務を切り替え
- **ナレッジベース** - カテゴリ階層、検索、Markdown対応
- **画像モーダル** - ズーム、パン操作
- **文字サイズ調整** - アクセシビリティ考慮

## 技術スタック

- Chrome Extension Manifest V3
- Vanilla JavaScript（ES Modules）
- CSS Variables

フレームワーク不使用で軽量かつ保守性を確保。
